STM32G474 Firmware

Upload stm32_firmware.bin → flash the weld controller via Katapult.
How this works: the file is uploaded to this Flask server, then forwarded to the device's HTTP flash endpoint (/ota or /stm32) on port 80. This is a separate service from the telemetry bridge on port 8888 — so flashing here will not interrupt a MobaXterm UART stream or the live Control/Monitor data.

Safety: the server refuses to flash an ELF file (it must be the raw .bin). Flashing an ELF previously bricked the STM32 because the header bytes land on the vector table.

Note: after a successful ESP32-P4 flash, the device reboots — the Control/Monitor pages will reconnect automatically once it is back up.
